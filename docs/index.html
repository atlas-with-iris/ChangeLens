Six steps. Fully deterministic.
Same input → same output.

No LLM inference. No cloud calls. Pure static analysis running locally on your machine.

01

Parse Diff

Unified diff → structured file changes with symbol extraction. Functions, classes, exports — not just file names.

→
02

Classify Surface

Categorizes each changed file: API · AUTH · CONFIG · SCHEMA · UTILITY · TEST · STYLE · BUILD · DOCS

→
03

Walk Graph

BFS traversal of your import/require graph. Finds every downstream consumer of every changed file.

→
04

Calculate Blast

Surface weight × consumer count × export changes → risk tier. Conservative by design — when in doubt, flag it.

→
05

Build Card

Assembles a 9-field Impact Card with evidence, consumers, surfaces, risk tier, and merge guidance.

→
06

Ship Result

Renders as a GitHub PR comment, JSON for CI, or rich terminal output with ANSI colors.

Up and running in 30 seconds.

CLI

$ git diff main | npx changelens

Pipe any git diff. Get an Impact Card. That's it.

# From a diff file $ npx changelens --diff changes.patch # JSON for CI pipelines $ git diff main | npx changelens --json

GitHub Action

# .github/workflows/changelens.yml name: ChangeLens on: [pull_request] jobs: impact: runs-on: ubuntu-latest permissions: pull-requests: write steps: - uses: actions/checkout@v4 - uses: atlas-with-iris/ChangeLens@v1 env: GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

Auto-posts an Impact Card on every PR. Updates on each push.
